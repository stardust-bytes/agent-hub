## What's Inside

### 🤝 Cowork — AI Assistant

Work alongside an AI agent that understands your documents and office data. Cowork mode provides:

- **File Management** — Browse, preview, and search documents (PDF, DOCX, XLSX), images, and markdown files inline
- **SSE Streaming** — Real-time agent responses with markdown rendering, tool calls, and plan execution
- **Context-aware Assistance** — Agent reads your documents, understands your data, and performs office tasks directly
- **Multi-step Plans** — Agent autonomously plans and executes complex workflows, shows progress, resumes after interruption

### 📝 Notes

Markdown notes CRUD with title and content. Quick capture and organization.

### 🔌 Connectors

Connect external services via OAuth:

- **Gmail** — Search, read, send, draft emails
- **Google Calendar** — List, create, update, check availability
- **Google Drive** — Search, read, list, upload files

### 🧰 Tools Registry

Fine-grained control over every tool the agent can use. Enable/disable individual tools, configure tool-specific settings, and control execution permissions.

### 📋 Plans

The agent can autonomously create execution plans from your requests:
- Multi-step plan generation
- Step-by-step execution with status tracking
- Resume interrupted plans
- Approve/reject individual steps

### ⏰ Scheduled Tasks

Cron-based automation for recurring AI tasks:
- Hourly, daily, weekly, or custom schedules
- Model and project path selection per schedule
- Execution logs with status tracking

### 📂 Agent Outputs

List and download files generated by the AI agent during your sessions. Organised per session with file size and timestamps.

### ⚙️ Settings

Central configuration panel:
- **Model Config** — Default model, embedding model, summary model
- **MCP Servers** — Enable/disable MCP tool servers
- **Google OAuth** — Configure Client ID/Secret for Gmail, Calendar, Drive

### 🧠 Memory

Long-term memory with auto-extraction:
- Types: User, Feedback, Project, Reference
- Auto-extracted from agent conversations
- Searchable, editable, deletable
- Injected into agent context for personalised responses

### 📊 Usage Tracking

Per-session token usage breakdown: prompt tokens, completion tokens, total tokens per model and provider.

### 🛡️ Permissions & YOLO Mode

Granular tool permission system:
- **Permission Modes** — Default, Accept Edits, Bypass All, Don't Ask, Auto (YOLO), Plan
- **Block Rules** — Control over interpreters, package runners, shells, network, git destructive, and more
- **Tool Approval** — Require manual confirmation for sensitive tools

## 🔒 100% Local First

Everything stays on your machine:

| Component | Data Store |
|---|---|
| **Chat History** | SQLite via Prisma |
| **Vector Index** | LanceDB (local file) |
| **AI Inference** | Ollama (localhost:11434) |
| **File Storage** | Local workspace directory |
| **Telemetry** | None — zero data leaves your machine |

No accounts, no cloud services, no data sharing. Your code and conversations never leave your network.

## Quick Start

### Prerequisites

- **Node.js 20+** — [nodejs.org](https://nodejs.org)
- **Ollama** (optional, for local AI) — [ollama.ai](https://ollama.ai)

### Option 1: npm (development)

```bash
# Backend
cd backend
npm install
npm run setup
npm run start:dev          # http://localhost:13596

# Frontend (separate terminal)
cd frontend
npm install
npm run dev                # http://localhost:17135
```

### Option 2: Single command (Windows)

```powershell
.\run.ps1
```

Opens both backend and frontend automatically. Add `-Docker` flag to use Docker instead.

### Option 3: Docker (recommended for production)

```bash
docker compose up --build
```

Opens at **http://localhost:17135**. The backend runs on port 13596 (internal). Requires Docker Desktop.

## Configuration

Copy `backend/.env.example` → `backend/.env`:

```env
DATABASE_URL=file:/app/data/dev.db
PORT=13596
OLLAMA_URL=http://localhost:11434
ACTIVE_PROVIDER=ollama
EMBED_MODEL=nomic-embed-text
SUMMARY_MODEL=llama3.2
```

Add AI providers via the UI (Settings → Providers):

| Provider | Type | Notes |
|---|---|---|
| Ollama | `ollama` | Local, requires Ollama running |
| OpenAI | `openai` | `baseUrl: https://api.openai.com/v1` |
| DeepSeek | `openai` | `baseUrl: https://api.deepseek.com/v1` |
| Any OpenAI-compatible | `openai` | Custom `baseUrl` |
