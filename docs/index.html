Local & Private
Everything, self-hosted

One app, a lot of capabilities.

Started as an AI chat. Became a workspace. Each piece runs locally against whatever endpoints you point it at.

💬

Chat & Agents

Multi-turn chat plus autonomous agents that plan, call tools, and work through complex tasks — all against your local or remote models.

🔧

Tools & MCP

Built-in tools (bash, files, web, memory, code execution) plus any MCP server you connect. Toggle per tool with granular permission control.

📚

RAG & Knowledge

Upload documents (.pdf, .docx, .xlsx) or watch a codebase directory. LanceDB-powered vector search makes your data accessible to the agent.

📧

Email & Calendar

AI-powered email summaries, style-matched draft replies, auto-tagging, and spam triage over IMAP/SMTP. Google Calendar integration for scheduling.

📊

Google Workspace

Connect Gmail, Calendar, Google Drive, and Google Sheets. The agent can read, write, and manage your data directly.

📅

Scheduled Tasks

Create cron-driven agent runs. The agent executes prompts on a schedule and logs results. Perfect for daily briefings, reports, and monitoring.

🧠

Persistent Memory

The assistant builds up persistent memory across all your conversations, getting more capable and context-aware over time.

🎯

Model Agnostic

Works with Ollama, OpenAI, DeepSeek, or any OpenAI-compatible API. Full provider management UI — add, configure, and switch models on the fly.

🔒

Private by Default

Runs entirely on your machine. No telemetry, no data leaves your network. Optional external integrations only when you explicitly enable them.

How it actually started

Built from a single prompt.

Agent Hub started as an experiment: could an AI agent build its own workspace? The project grew from a single prompt into a full-featured, local-first AI workspace with chat, autonomous agents, tools, RAG, and deep integrations.

The idea was simple — running local AI should feel empowering, not limiting. Every tool, every connector, every feature was built to give you and your agents more capability without compromising privacy. The more context you give it, the more useful it becomes.

Which is exactly why it's self-hosted: all that context stays on your machine.
